# コミットメッセージ作成

`$ARGUMENTS` で渡された変更内容または差分の要約から、コミットメッセージ案を作成してください。

## 引数

- 基本は `$ARGUMENTS` に変更内容が分かるファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を変更内容の説明文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 対象ファイルがあれば確認し、コミットの主目的を 1 つに絞る
2. `feat` `fix` `docs` `refactor` `test` `chore` から最も近い `type` を 1 つ選ぶ
3. `<type>: <要約>` 形式で、日本語の短い要約を作る
4. 1 コミット 1 目的に収まらない場合は、分割案も添える

## 出力フォーマット

- 推奨コミットメッセージ
- 判定理由
- 分割が必要な場合は代替案
