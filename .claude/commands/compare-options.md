# 比較検討

`$ARGUMENTS` で渡された対象ファイルまたはテーマについて、選択肢を比較してください。

## 引数

- 基本は `$ARGUMENTS` に比較元のファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を比較テーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 比較テーマを 1 文で定義する
2. 対象ファイルがあれば確認し、比較対象を 3 つ以内に整理する
3. 比較軸を定義する
4. 各選択肢を同じ粒度で評価する
5. 推奨案と理由を明記する
6. 採用時のリスクと緩和策をまとめる

## 出力先

- 持ち込み先リポジトリの `docs/10_research/` または `docs/40_decisions/` に保存する前提で出力する

## 出力フォーマット

- 比較テーマ
- 比較対象
- 比較軸
- 比較表
- 推奨案
- 採用理由
- リスクと緩和策
- 要確認事項
