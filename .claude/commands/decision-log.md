# 意思決定ログ作成

`$ARGUMENTS` で渡された対象ファイルまたはテーマについて、横断的な意思決定ログを作成してください。

## 引数

- 基本は `$ARGUMENTS` に判断材料のファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を決定テーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 決定事項を 1 文で定義する
2. 背景と選択肢を整理する
3. 最終決定と採用理由を明記する
4. 影響範囲、リスク、フォローアップを整理する

## 出力先

- 持ち込み先リポジトリの `docs/40_decisions/` 配下に保存する前提で Markdown 形式で出力する

## 出力フォーマット

- 決定事項
- 決定日
- ステータス
- 背景
- 選択肢
- 決定
- 理由
- 影響範囲
- リスクと対応
- 要確認事項
- 関連ドキュメント
