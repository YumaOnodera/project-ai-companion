# 連携手順書作成

`$ARGUMENTS` で渡された対象ファイルまたはテーマについて、外部サービス連携手順を作成してください。

## 引数

- 基本は `$ARGUMENTS` に対象ファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を連携テーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 連携対象と利用目的を明記する
2. 前提条件、必要権限、事前確認事項を整理する
3. 設定手順を実行順で記載する
4. 動作確認項目と運用上の注意点をまとめる

## 出力先

- 持ち込み先リポジトリの `docs/30_integrations/` 配下に保存する前提で Markdown 形式で出力する

## 出力フォーマット

- タイトル
- 対象
- 前提
- 事前確認
- 設定手順
- 動作確認
- 運用上の注意
- 要確認事項
- 参照元
