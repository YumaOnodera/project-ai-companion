# ナレッジ化

`$ARGUMENTS` で渡された対象ファイルまたはテーマで得た内容を、再利用可能なナレッジに変換してください。

## 引数

- 基本は `$ARGUMENTS` に元情報のファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` をナレッジ化対象のテーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 元情報の要点を抽出する
2. 再利用しやすい手順または判断基準に再構成する
3. 適用条件と非適用条件を明記する
4. 注意事項と関連ドキュメントを整理する

## 出力先

- 持ち込み先リポジトリの `docs/50_knowledge/` に保存する前提で Markdown 形式で出力する

## 出力フォーマット

- タイトル
- 更新日
- 概要
- 適用条件
- 非適用条件
- 手順 / 判断基準
- 注意事項
- 関連ドキュメント
