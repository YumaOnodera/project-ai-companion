# 調査メモ作成

`$ARGUMENTS` で渡された対象ファイルまたはテーマについて、調査メモを作成してください。

## 引数

- 基本は `$ARGUMENTS` に対象ファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を調査テーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 調査テーマと調査目的を明記する
2. 前提条件、対象範囲、確認日を整理する
3. 調査結果は事実に限定してまとめる
4. 解釈や仮説は `考察（推測）` として分離する
5. 参照元と要確認事項を残す

## 出力先

- 持ち込み先リポジトリの `docs/10_research/` 配下に保存する前提で Markdown 形式で出力する

## 出力フォーマット

- 調査テーマ
- 調査目的
- 前提
- 調査結果（事実）
- 考察（推測）
- 要確認事項
- 参照元
- 次アクション
