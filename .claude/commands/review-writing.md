# 文書表現レビュー

`$ARGUMENTS` で渡された Markdown について、このリポジトリの文体に合っているか確認し、改善案を提示してください。

## 引数

- 基本は `$ARGUMENTS` に対象ファイルパスを 1 つ以上渡す
- 対象は `docs/**/*.md` `README.md` `.claude/**/*.md` を想定する
- 指定されたファイルがある場合は、その内容を最優先で確認する
- ファイル指定がない場合のみ、`$ARGUMENTS` をレビュー対象テキストとして扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 冗長表現、誇張表現、AI っぽい定型句がないか確認する
2. 推測や未確認事項に `要確認` が付いているか確認する
3. 見出し、箇条書き、表現の粒度がそろっているか確認する
4. 問題箇所ごとに、短く自然な修正文を示す

## 出力フォーマット

- 対象
- 指摘事項
- 改善案
- 要確認事項
