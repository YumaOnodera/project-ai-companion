# 作業手順書作成

`$ARGUMENTS` で渡された対象ファイルまたはテーマについて、定型作業手順を作成してください。

## 引数

- 基本は `$ARGUMENTS` に対象ファイルパスを 1 つ以上渡す
- 指定されたファイルがある場合は、その内容を最優先で参照する
- ファイル指定がない場合のみ、`$ARGUMENTS` を手順化したいテーマ文字列として扱う
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 手順

1. 手順の目的と対象読者を明記する
2. 前提条件、必要な権限、準備物を整理する
3. 実行手順を順番に記載する
4. 確認ポイントと注意事項を分けて書く

## 出力先

- 持ち込み先リポジトリの `docs/20_workflows/` 配下に保存する前提で Markdown 形式で出力する

## 出力フォーマット

- タイトル
- 目的
- 対象
- 前提
- 手順
- 確認ポイント
- 注意事項
- 関連ドキュメント
