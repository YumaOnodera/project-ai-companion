# AI Context Note

このスキルは、AI エージェントに渡す参照方針や作業ルールを整理する時に使う。

## Use When

- ユーザーが AI 作業時の前提、参照優先順位、禁止事項の整理を求めている
- `docs/90_ai_context/` に置く前提の Markdown を作りたい
- 個別作業より上位の指示ルールを明文化したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 何の作業で使うコンテキストかを `目的` に書く。
2. どこまで適用するかを `適用範囲` に定義する。
3. 参照すべきドキュメントや正本の優先順位を整理する。
4. 守るべき `指示ルール` と `禁止事項` を分ける。
5. 見直し条件や更新契機を `更新条件` にまとめる。
6. 依存する正本ドキュメントを `関連ドキュメント` に残す。

## Output Format

以下の見出し順を基本とする。

- タイトル
- 目的
- 適用範囲
- 参照優先順位
- 指示ルール
- 禁止事項
- 更新条件
- 関連ドキュメント

## Writing Rules

- 一時的な依頼文ではなく、繰り返し使う前提の方針として書く。
- 参照優先順位は上から順に読める形で明示する。
- 守るべきことと避けるべきことを同じ箇条書きに混在させない。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/90_ai_context/` を第一候補にする。
