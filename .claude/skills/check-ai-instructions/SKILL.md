# Check AI Instructions

このスキルは、AI エージェント向けの指示文や運用ルールを見直す時に使う。

## Use When

- ユーザーが AI 指示文、運用指示、ルール文書のレビューを求めている
- 曖昧表現、重複、矛盾、責務の衝突を洗い出したい
- 修正案を最小差分で整理したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 想定対象は `AGENTS.md` `.claude/CLAUDE.md` `.claude/commands/*.md` `docs/90_ai_context/*.md` とする。
- ファイル指定がない場合のみ、会話内の指示文をレビュー対象として扱う。
- 複数ファイルがある場合は、適用範囲の重なりと優先順位も合わせて確認する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 対象文書の目的と適用範囲を確認する。
2. 曖昧表現、重複指示、優先順位の不明瞭さ、矛盾を抽出する。
3. 指示の責務が別文書と重複していないか確認する。
4. 参照優先順位がある場合は、上から順に読める形か確認する。
5. 問題ごとに、どこをどう直すべきか最小差分で提案する。
6. 不確実な点は `要確認` として分離する。

## Output Format

以下の項目順を基本とする。

- 対象
- レビュー結果
- 問題箇所
- 修正方針
- 要確認事項

## Writing Rules

- 問題の種類を `曖昧` `重複` `矛盾` `優先順位不明` などで明示する。
- 問題指摘だけで終わらせず、修正方針まで添える。
- 文書の役割が重複している場合は、寄せ先の正本も示す。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
