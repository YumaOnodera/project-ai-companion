# Commit Message

このスキルは、変更内容に応じたコミットメッセージ案を規約準拠で作る時に使う。

## Use When

- ユーザーがコミットメッセージ案の作成を求めている
- 変更差分に対して適切な `type` を選びたい
- このリポジトリの規約に沿って日本語要約を付けたい

## Workflow

1. 変更内容を読み、コミットの主目的を 1 つに絞る。
2. 規約の `type` 一覧から最も近いものを 1 つ選ぶ。
3. 要約は日本語で、変更の目的が分かる短文にする。
4. 生成物と手編集が混在している場合は、コミット分割が必要か確認する。
5. 1コミット1目的に収まらない場合は、分割案も添える。

## Commit Rules

コミットメッセージは次の形式を使う。

- `<type>: <要約>`

使用する `type` は以下。

- `feat`: 機能追加
- `fix`: バグ修正
- `docs`: ドキュメント変更
- `refactor`: 挙動変更なしの内部改善
- `test`: テスト追加・修正
- `chore`: 設定・運用・依存更新など

## Output Format

以下を基本とする。

- 推奨コミットメッセージ
- 判定理由
- 分割が必要な場合は代替案

## Writing Rules

- 要約は日本語で書く。
- 変更内容の羅列ではなく、主目的が伝わる要約を優先する。
- 規約上あいまいな場合は、候補を 2 件までに絞って理由を添える。
- 差分が複数目的にまたがる場合は、無理に 1 件へ押し込めず分割案を出す。
