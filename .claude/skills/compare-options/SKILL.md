# Compare Options

このスキルは、複数案の比較検討を文書化する時に使う。

## Use When

- ユーザーが案比較、選定、技術選択、導入判断を求めている
- 2 から 3 案を同じ粒度で評価したい
- 推奨案と採用理由、採用時の注意点まで残したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 比較テーマを 1 文で定義する。
2. 比較対象は最大 3 つに絞る。
3. 比較軸を定義する。
4. 全選択肢を同じ粒度で評価する。
5. 推奨案を 1 つ示し、理由を簡潔に述べる。
6. 採用時のリスクと緩和策を整理する。
7. 未確定情報は `要確認事項` として残す。

## Default Comparison Axes

必要に応じて次を使う。

- コスト
- 導入難易度
- 運用負荷
- 拡張性
- リスク

## Output Format

以下の見出し順を基本とする。

- 比較テーマ
- 比較対象
- 比較軸
- 比較表
- 推奨案
- 採用理由
- リスクと緩和策
- 要確認事項

## Writing Rules

- 比較表だけで終わらせず、最終判断を必ず書く。
- 評価基準が曖昧な場合は前提条件を補う。
- 根拠が弱い比較は断定せず、確認方法を添える。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は、検討メモなら `docs/10_research/`、意思決定記録なら `docs/40_decisions/` を候補にする。
