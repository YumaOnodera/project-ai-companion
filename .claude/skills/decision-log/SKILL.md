# Decision Log

このスキルは、後から判断理由を追える形で意思決定を記録する時に使う。

## Use When

- ユーザーが運用判断、方針決定、ADR の記録を求めている
- `docs/40_decisions/` に置く前提の Markdown を作りたい
- 比較結果だけでなく、最終決定と影響範囲を明示したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. `決定事項` を 1 文で定義する。
2. `決定日` と `ステータス` を明記する。
3. 背景と意思決定が必要になった理由を整理する。
4. 選択肢は最大 3 つまでに絞る。
5. 採用した決定と理由を簡潔に記す。
6. 影響範囲、リスク、フォローアップを整理する。
7. 未確定事項は `要確認事項` に残す。

## Output Format

以下の見出し順を基本とする。

- 決定事項
- 決定日
- ステータス
- 背景
- 選択肢
- 決定
- 理由
- 影響範囲
- リスクと対応
- 要確認事項
- 関連ドキュメント

## Writing Rules

- 選択肢比較だけで終わらせず、最終決定を必ず書く。
- 変更される可能性がある場合は `ステータス` に `draft` `accepted` `superseded` などを入れる。
- 背景と理由は区別し、読み返した時に判断根拠が追えるようにする。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/40_decisions/` を第一候補にする。
