# Integration Guide

このスキルは、外部サービス連携の導入手順や運用条件を整理する時に使う。

## Use When

- ユーザーが SaaS、API、MCP、外部ツール連携の手順整理を求めている
- `docs/30_integrations/` に置く前提の Markdown を作りたい
- 導入手順だけでなく、事前確認や動作確認、注意点まで残したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 連携対象と利用目的を短く定義する。
2. 確認日、前提条件、必要権限を整理する。
3. 導入前の `事前確認` をまとめる。
4. 設定や接続の手順を実行順で記載する。
5. 導入後の `動作確認` を列挙する。
6. 運用上の制約や権限差分は `運用上の注意` に分ける。
7. 未確定事項は `要確認事項` に残し、参照元を付ける。

## Output Format

以下の見出し順を基本とする。

- タイトル
- 対象
- 前提
- 事前確認
- 設定手順
- 動作確認
- 運用上の注意
- 要確認事項
- 参照元

## Writing Rules

- 公式情報がある場合は参照元を必ず明記する。
- 手順と判断メモを混在させない。
- 料金、権限、提供条件のような変わりやすい情報には確認日を付ける。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/30_integrations/` を第一候補にする。
