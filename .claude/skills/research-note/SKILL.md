# 調査メモ作成 skill

Claude Code の定型作業を `/research-note` で呼ぶための skill。

## 使う場面

- 調査結果をあとで再利用できる形で残したい
- `docs/10_research/` 向けに Markdown を作りたい
- 事実と推測を混在させずに整理したい

## 入力の扱い

- 指定ファイルがある場合は最優先で使う
- ファイル指定がない時だけ、会話中のテーマを元情報にする
- 明示のない `docs/99_tmp/` 全体は自動走査しない

## 進め方

1. 調査テーマと調査目的を明記する
2. 前提条件、対象範囲、確認日を整理する
3. 調査結果は `事実` に限定する
4. 解釈は `考察（推測）` に分離する
5. 参照元と `要確認事項` を残す

## 出力

- 調査テーマ
- 調査目的
- 前提
- 調査結果（事実）
- 考察（推測）
- 要確認事項
- 参照元
- 次アクション
