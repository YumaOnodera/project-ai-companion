# Research Note

このスキルは、テーマ調査を再利用しやすいメモへ整理する時に使う。

## Use When

- ユーザーが調査メモ、調査ノート、リサーチ結果の整理を求めている
- `docs/10_research/` に置く前提の Markdown を作りたい
- 事実と推測を分け、参照元と未確定事項を残したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 調査テーマと調査目的を明記する。
2. 前提条件、対象範囲、確認できた日付を整理する。
3. 調査結果は「事実」に限定してまとめる。
4. 解釈や仮説は「考察（推測）」として分離する。
5. 未確認の点は `要確認` と明記する。
6. 参照元は URL や資料名を列挙する。
7. 次に取るべきアクションを 2 から 4 件程度で提案する。

## Output Format

以下の見出し順を基本とする。

- 調査テーマ
- 調査目的
- 前提
- 調査結果（事実）
- 考察（推測）
- 要確認事項
- 参照元
- 次アクション

## Writing Rules

- 事実と推測を混在させない。
- 推測を含む場合は断定せず、根拠が弱ければ `要確認` を付ける。
- 日付依存の情報は、可能なら確認日または更新日を明記する。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/10_research/` を第一候補にする。
