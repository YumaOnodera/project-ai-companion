# Check Fact

このスキルは、文書やメモに書かれた内容が `workspace` 内の正本と一致しているか確認する時に使う。

## Use When

- ユーザーが事実確認、真偽確認、正本照合を求めている
- 対象文書の内容が正しいかを根拠付きで確認したい
- 誤りがあれば修正案まで整理したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- ファイル指定がない場合のみ、会話内の記述や主張を検証対象として扱う。
- 複数ファイルがある場合は、主張を分解して 1 件ずつ確認する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Source Priority

確認は次の順で行う。

1. `AGENTS.md`
2. `.claude/CLAUDE.md` `.claude/settings.json` `.claude/commands/`
3. `docs/90_ai_context/`
4. 対象ディレクトリの正本文書
5. `docs/99_tmp/` の入力メモ

## Workflow

1. 検証対象の主張を 1 件ずつ分解する。
2. `workspace` 内の正本を優先して確認する。
3. `正しい` `誤り` `一部正しい` `判断不可` の 4 区分で判定する。
4. 根拠は必ずファイルパスと該当箇所付きで示す。
5. 誤りや不足がある場合は、正しい記述案または修正方針を添える。
6. 推測が混ざる場合は `要確認` を明記する。

## Output Format

以下の項目順を基本とする。

- 検証対象
- 結論
- 根拠
- 修正案
- 要確認事項

## Writing Rules

- 正誤だけで終わらせず、必ず根拠を示す。
- 根拠が弱い場合は断定せず `判断不可` または `要確認` を使う。
- 外部情報ではなく、まず `workspace` 内の正本を優先する。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
