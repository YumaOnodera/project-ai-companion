# Knowledge Capture

このスキルは、会話、調査、実作業の結果を再利用可能なナレッジへ整形する時に使う。

## Use When

- ユーザーが知見の整理、ナレッジ化、運用知見の文書化を求めている
- 一度限りのメモではなく、再利用前提の形にしたい
- `docs/50_knowledge/` 向けの Markdown を作りたい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 元情報から要点だけを抽出する。
2. 手順または判断基準として再構成する。
3. その知見が有効な条件を `適用条件` に書く。
4. 使うべきでない条件を `非適用条件` に書く。
5. 失敗しやすい点や誤用しやすい点を `注意事項` にまとめる。
6. 冒頭に更新日を入れる。
7. 関連する正本ドキュメントがあれば `関連ドキュメント` に寄せる。

## Output Format

以下の見出し順を基本とする。

- タイトル
- 更新日
- 概要
- 適用条件
- 非適用条件
- 手順 / 判断基準
- 注意事項
- 関連ドキュメント

## Writing Rules

- 元の文脈依存の表現を減らし、再利用時にも読める形にする。
- 参照先に正本がある場合は重複記述を避け、リンクや参照関係を明記する。
- 推測を含む場合は `要確認` を付ける。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/50_knowledge/` を第一候補にする。
