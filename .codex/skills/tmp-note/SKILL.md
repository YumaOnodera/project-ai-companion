# Tmp Note

このスキルは、まだ正式な保存先が決まっていない途中メモを素早く残す時に使う。

## Use When

- ユーザーが下書き、壁打ちメモ、途中整理を求めている
- `docs/99_tmp/` に置く前提の Markdown を作りたい
- 後で `10_research` `20_workflows` `40_decisions` `50_knowledge` へ移す前の材料を残したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. メモの `目的` と作業日を明記する。
2. 現時点の情報を `入力メモ` として残す。
3. 解釈や仮説は `仮説` に分離する。
4. 不明点は `要確認事項` に残す。
5. 次の打ち手を `次アクション` に書く。
6. 正式化先の候補を `移管先候補` に示す。

## Output Format

以下の見出し順を基本とする。

- タイトル
- 作業日
- 目的
- 入力メモ
- 仮説
- 要確認事項
- 次アクション
- 移管先候補

## Writing Rules

- きれいにまとめ過ぎず、後で整理できる粒度を優先する。
- 事実と仮説を混在させない。
- 長期保管前提の表現は避け、必要なら正式ドキュメントへ移す前提を残す。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/99_tmp/` を第一候補にする。
