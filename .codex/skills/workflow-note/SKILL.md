# Workflow Note

このスキルは、定型作業を再実行しやすい手順書へ整理する時に使う。

## Use When

- ユーザーが手順書、運用フロー、作業の標準化を求めている
- `docs/20_workflows/` に置く前提の Markdown を作りたい
- 目的、前提、実行手順、確認ポイントを一式で残したい

## Input Handling

- ユーザーが対象ファイルを明示した場合は、そのファイルを元情報として最優先で読む。
- 複数ファイルがある場合は、共通点と差分を整理してから出力へ反映する。
- 明示のない `docs/99_tmp/` 全体を自動走査せず、指定されたファイルか会話内の入力だけを使う。

## Workflow

1. 手順の目的と対象読者を明記する。
2. 前提条件、必要な権限、準備物を整理する。
3. 手順は実行順で並べる。
4. 正常終了の確認ポイントを分けて書く。
5. 失敗しやすい点や例外は `注意事項` に寄せる。
6. 他に正本がある場合は `関連ドキュメント` でつなぐ。

## Output Format

以下の見出し順を基本とする。

- タイトル
- 目的
- 対象
- 前提
- 手順
- 確認ポイント
- 注意事項
- 関連ドキュメント

## Writing Rules

- 単なる作業ログではなく、再利用可能な手順として書く。
- 手順と注意事項を混在させない。
- 実行条件が変わりやすい箇所は `要確認` を付ける。
- 対象ファイルがある場合は、その内容を正本の入力として優先する。
- このリポジトリで保存先まで求められた場合は `docs/20_workflows/` を第一候補にする。
