# Project Instructions

## Project Overview

- Project name: project-ai-companion
- Purpose: Manage reusable AI instructions, editor settings, and documentation rules that can be brought into existing business-system repositories
- This repository is not an application template and is not used to manage project-specific requirements, specifications, designs, or implementation details

## Communication

- Respond in Japanese unless the target file or user request requires another language
- Keep responses concise and skip obvious explanations
- Show a short plan before substantial work

## Working Policy

- Prefer reusable, cross-project guidance over project-specific rules
- Keep copied content minimal and reorganize it for this repository's role instead of duplicating source material as-is
- Mark uncertain information as `要確認`

## Documentation Policy

- Canonical documents live under `docs/`
- Temporary notes go under `docs/99_tmp/`
- Avoid duplicate descriptions when a source of truth already exists
- Keep Markdown headings unique within a file
- Use compact Markdown tables like `| a | b |`

## Prohibited

- Do not commit secrets, keys, or personal information
- Do not store customer-specific or company-specific confidential information
- Do not leave unsupported assertions in documentation
