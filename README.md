# project-ai-companion

既存の業務システム案件へ後から持ち込める、AI 設定、最小限の文書運用ルール、補助資料の共通リポジトリです。
新規アプリ雛形ではなく、横断再利用したい設定と資料だけを管理します。

## このリポジトリで扱うもの

- `AGENTS.md` と Codex / Claude Code 向け設定の叩き台
- `docs/` 配下で再利用する AI 文脈と導入テンプレート
- 持ち込み先リポジトリで使う `.claude/skills/` の雛形
- Codex / Claude で再利用する文書化系 skill の draft
- Markdown 運用、AI 依頼、ナレッジ整理のための共通ルール
- 既存案件へコピーしやすい最小限の editor / lint 設定

## 扱わないもの

- 案件固有の要件、仕様、設計、実装
- 顧客情報、秘密情報、社内固有情報
- 特定案件専用のコマンドやフレームワーク実装規約
- 時変情報を前提にした比較一覧の恒久保存

## 現在の主な構成

```text
.
├── AGENTS.md
├── README.md
├── .claude/
├── .codex/
├── .markdownlint.jsonc
├── .vscode/
├── docs/
└── package.json
```

## 使い方

1. このリポジトリから必要な設定や資料だけを対象案件へコピーする
2. 案件側で `AGENTS.md` `.claude/CLAUDE.md` `docs/90_ai_context/` を案件文脈に合わせて調整する
3. 案件で再利用価値が確認できた改善だけを、このリポジトリへ戻す

## 参照開始点

- 全体像: `docs/00_overview.md`
- 保存先ルール: `docs/README.md`
- AI 文脈: `docs/90_ai_context/`

## 運用原則

- 横断再利用できる最小限の内容だけを残す
- 既存の正本がある場合は重複記述を避ける
- 未確定事項は `要確認` と明記する
- コピー元の文書をそのまま複製せず、このリポジトリの役割に合わせて再構成する
