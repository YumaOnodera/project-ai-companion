# project-ai-companion

既存の業務システム案件で再利用するための、AI 設定・文書運用ルール・開発ガードレールの共通設定リポジトリです。
新規アプリの雛形ではなく、既存リポジトリへ後から持ち込むための最小セットを管理します。

## 向いているケース

- 既存案件に AI 向けの指示や運用ルールを追加したい
- 案件ごとに `AGENTS.md` や AI 設定を作り直す手間を減らしたい
- 個別案件のルールと横断運用ルールを分けて管理したい

## 目的

- 案件ごとに AI 設定を作り直す手間を減らす
- 業務システム開発で使いやすいガードレールをそろえる
- プロジェクト固有情報と横断運用ルールを分離する
- 既存案件へ安全に AI 支援設定を持ち込めるようにする

## このリポジトリの位置づけ

- 新規業務システムの雛形ではない
- 基本は、既存案件リポジトリへ必要なファイルだけをコピーして使う
- リポジトリごと `clone` する場合は、参照用ディレクトリまたは `submodule` `subtree` などの運用を前提にする
- 案件固有の要件、仕様、設計、実装は各案件リポジトリ側で管理する
- このリポジトリでは、横断的に再利用したい AI 設定と運用ルールだけを管理する

## 含めるもの

- `AGENTS.md` の雛形
- `.codex/` の雛形
- `.claude/` の雛形
- `docs/` の最小テンプレート
- AI への依頼テンプレート
- 導入手順と更新ルール

## 含めないもの

- 案件固有の要件、仕様、設計
- 顧客情報、秘密情報、社内固有情報
- 特定案件専用のコマンドや運用ルール
- 特定フレームワーク前提の実装規約

## 想定ディレクトリ

```text
.
├── README.md
├── AGENTS.md
├── .codex/
├── .claude/
└── docs/
```

## 使い方

1. このリポジトリを任意の場所へ clone する
2. 案件リポジトリへ必要なファイルやディレクトリをコピーする
3. 案件固有ルールに合わせて `AGENTS.md` と AI 設定を調整する
4. 案件横断で再利用したい改善だけを、このリポジトリへ戻す

## 導入対象の例

- Codex 向けのプロジェクト指示
- Claude Code 向けのプロジェクト指示
- AI に渡す参照優先順位
- ドキュメント運用ルール
- AI 依頼テンプレート
- レビュー観点と確認手順

## 運用ルール

- 横断的な運用知見だけをここで管理する
- 案件固有情報は案件リポジトリ側で管理する
- 未確定事項は `要確認` と明記する
- 秘匿情報は保存しない
- コピー元の `workspace` や他リポジトリと役割が重複しないように保つ

## 初期導入候補

- AI 指示: `AGENTS.md`, `.claude/CLAUDE.md`
- Codex 雛形: `.codex/config.toml.example`, `.codex/skills/`
- 文書運用: `docs/README.md`, `docs/90_ai_context/`
- ガイドライン: AI 依頼テンプレート、レビュー観点、更新フロー

## 今後の方針

- 最初は最小構成で運用を始める
- 案件で再利用が確認できたものだけを追加する
- 流用しにくい案件固有ルールは持ち込まない
