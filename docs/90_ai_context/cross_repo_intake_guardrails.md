# Cross-Repo Intake Guardrails

## 目的

`workspace` や既存案件リポジトリから、このリポジトリへ設定や資料を取り込む時の境界をそろえる。

## 適用範囲

- 他リポジトリの `docs/` や AI 向け指示文を確認し、このリポジトリへ反映する作業
- 文書追加、統合、移管、重複解消の判断

## 参照優先順位

1. `docs/90_ai_context/repository_documentation_context.md`
2. `README.md`
3. `docs/README.md`
4. `docs/90_ai_context/agent_instruction_source_map.md`
5. 取り込み元リポジトリの `AGENTS.md` `docs/README.md` `docs/90_ai_context/*`
6. このリポジトリの既存正本文書

## 指示ルール

- 最初に、その内容が横断再利用知見か、元リポジトリ固有の前提かを判定する
- 他リポジトリからは、再利用できる判断基準、参照導線、運用ルールだけを抽出する
- 既存文書に同じ役割の正本がある場合は、新規作成より既存更新を優先する
- `docs/90_ai_context` には AI が着手前に合わせるべき前提と導入テンプレートだけを書く
- 元文書をそのまま複製せず、このリポジトリの役割に合わせて再構成する
- 反映後は、確定した内容と `要確認` のまま残す内容を分けて示す

## 禁止事項

- 実装プロジェクト固有の要件、仕様、設計、コード方針を正本として持ち込まない
- 役割が重なるテンプレートや方針文書を増やしすぎない
- 根拠や適用範囲を確認せずに横断ルール化しない

## 更新条件

- このリポジトリと取り込み元の責務分担が変わった時
- 他リポジトリからの知見取り込み基準が変わった時
