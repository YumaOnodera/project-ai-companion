# Docs Structure

この `docs/` は、既存の業務システム案件リポジトリへ持ち込める AI 文脈と導入テンプレートを管理するためのディレクトリです。

## 目的

- AI 指示や参照優先順位を再利用しやすくする
- 導入時に使うテンプレートを最小構成で管理する
- 案件固有情報と横断知見を分離する

## ディレクトリ構成

- `90_ai_context`: AI 作業時の参照方針、導入テンプレート、ガードレール
- `99_tmp`: 一時メモ

## 参照開始点

- 全体像: `docs/00_overview.md`
- AI 文脈: `docs/90_ai_context/repository_documentation_context.md`

## 運用ルール

- 案件固有の内容は各案件リポジトリ側で管理する
- 未確定事項は `要確認` と明記する
- 同じ役割の正本がある場合は重複記述を避ける
- 一時メモは `99_tmp` に置き、整理後に正式ディレクトリへ移す
